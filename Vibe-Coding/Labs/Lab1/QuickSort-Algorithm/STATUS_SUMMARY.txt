🔑 5 KEY OPTIMIZATIONS
═══════════════════════════════════════════════════════════════════════════════════

1. HOARE PARTITION ⭐⭐⭐
   Impact: 75% fewer swaps
   Method: Scans from both ends, swaps groups
   Result: Memory operations reduced significantly

2. MEDIAN-OF-THREE PIVOT ⭐⭐⭐
   Impact: Avoids O(n²) worst-case
   Method: Selects median of first, middle, last
   Result: Works well with sorted/reverse-sorted data

3. INSERTION SORT FOR SMALL ARRAYS ⭐⭐
   Impact: 20-30% overall improvement
   Method: Switch to InsertionSort when n < 10
   Result: Eliminates QuickSort overhead

4. TAIL RECURSION OPTIMIZATION ⭐⭐⭐
   Impact: O(log n) stack depth guaranteed
   Method: Always recurse on smaller partition first
   Result: No stack overflow risk

5. ITERATIVE IMPLEMENTATION ⭐⭐
   Impact: No recursion function call overhead
   Method: Uses explicit Stack<(int,int)>
   Result: Predictable memory, same performance
